    Introduction

    Welcome to the Hike CMS API! You can use our API to access CMS API endpoints, which can be used create new articles.

    We have language bindings in Shell! You can view code examples in the dark area to the right.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "api_endpoint_here"
-H "api_key: hipartner"

    Make sure to replace hipartner with your API key.

    Hike CMS uses API keys to allow access to the API.

    Hike CMS expects for the API key to be included in all API requests to the server in a header that looks like the following:

    api_key: hipartner

    You must replace hipartner with your personal API key.

    Articles

    Create Article

    Terminology

    1. Caption : Caption for the article
    2. Category : To be fetched from get Categories API
    3. Subcategory : To be fetched from get Subcategory by category API
    4. Cover : This contains the asset on which the user will land on opening the microapp.
    5. Post : This is shown to user when user swipes the cover page.
    curl -XPOST "https://api-creators.hike.in/v1/articles" -H "api_key: hipartner" -d \

    {
  "caption":"Hi there",
  "category":"Movies",
  "subCategory":"Bollywood",
  "cover": {
    "items": [{
      "content": {
        "size": 650299,
        "srcUrl": "https://cdn.xyz.com/anj/hike/323689801.gif",
        "format": "gif"
      },
      "type": "IMAGE"
    }],
    "type": "NORMAL"
  },
  "post": {
    "items": [{
      "content": {
        "size": 650299,
        "srcUrl": "https://cdn.xyz.com/anj/hike/323689801.mp4",
        "format": "mp4"
      },
      "type": "VIDEO"
    }],
    "type": "NORMAL"
  }
}

    This endpoint create an article.

    HTTP Request

    POST https://api-creators.hike.in/v1/articles

    Request Parameters

    Content Type Mandatory Fields
    IMAGE srcUrl
    VIDEO srcUrl , thumbnailUrl
    GIF srcUrl
    HTML body
    Cover/Post Type NORMAL


    Parameter Type Description Mandatory
    caption String Caption for the article true
    category String Category for the article fetched from get categories API true
    subCategory String Subcategory for the article fetched from get sub categories for categoryId API true
    cover Object Contains data for article cover true
    cover.type String Type of cover (NORMAL) true
    cover.items Object Array Contains data related to cover/post true
    cover.items.type String Type of cover/post (IMAGE/VIDEO/HTML) true
    cover.items.content.srcUrl String URL for the asset see above table
    cover.items.content.format String format of asset false
    cover.items.content.size String Size of the asset false
    cover.items.content.body String text body for content see above table
    cover.items.content.thumbnailUrl String thumbnailUrl for content see above table

    Example Response

    { “successMessage”: “Article successfully added” “articleId”: 17 } 
    {
 "successMessage": "Article successfully added",
 "articleId": 17
}

    Categories

    Get Categories

    For creating articles , sending category and subcategory are mandatory.

    curl "https://api-creators.hike.in/v1/categories" -H "api_key: hipartner"

    HTTP Request

    GET https://api-creators.hike.in/v1/categories

    {

    "categories": [
      {
        "id": "1",
        "name": "Movies"
      },
      {
        "id": "5",
        "name": "Sports"
      }
    ]
}

    Get subcategory by category id

    For creating articles , sending category and subcategory are mandatory.

    curl "https://api-creators.hike.in/v1/categories/:categoryId/subcategory" \
-H "api_key: hipartner"

    HTTP Request

    GET https://api-creators.hike.in/v1/categories/:categoryId/subcategory

    Request Parameters

    Parameter Type Description Mandatory
    categoryId integer Category Id to fetch subcategory true 
    {
  "categoryId": "1",
  "subCategories": [
    {
      "id": "2",
      "name": "Hollywood"
    },
    {
      "id": "3",
      "name": "Bollywood"
    },
    {
      "id": "4",
      "name": "Others"
    }
  ]
}

    Errors

    The CMS API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – Insufficient permissions
    404 Not Found – Not found
    405 Method Not Allowed – An invalid method
    406 Not Acceptable – You requested a format that isn’t json
    429 Too Many Requests – Rate limit exceeded
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.
